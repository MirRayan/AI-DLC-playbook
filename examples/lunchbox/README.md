# Lunchbox Reference — Answer Key

This folder is the completed reference solution for the [Hands-On Walkthrough](../../00-hands-on-walkthrough.md). It is what your `lunchbox-architecture` folder should roughly look like after the final phase.

**How to use it:** do the walkthrough yourself first, then compare your file against the matching reference **after each checkpoint**. Your wording will differ — that's fine and expected. What should match is the *shape*: the sections, the facts (5-person team, 11:00 close, 2-week timeline), the IDs, and the traceability between documents. Don't copy these files instead of doing the exercise; the learning is in the doing.

One difference to know: the reference `PROJECT-INDEX.md` shows its **final state** (end of Phase 6). During earlier phases, your index will list fewer documents — each phase's checkpoint tells you what it should contain at that point.

| Reference file | Walkthrough phase | Real path in a context repo |
|----------------|-------------------|------------------------------|
| [PROJECT-CONTEXT.md](PROJECT-CONTEXT.md) | Phase 0 | `docs/00-project-context/PROJECT-CONTEXT.md` |
| [PROJECT-INDEX.md](PROJECT-INDEX.md) | Phase 0, updated every phase | `PROJECT-INDEX.md` (repo root) |
| [CONCEPT-NOTE.md](CONCEPT-NOTE.md) | Phase 1 | `docs/01-concept/CONCEPT-NOTE.md` |
| [SPECIFICATION.md](SPECIFICATION.md) | Phase 2 | `docs/02-specification/SPECIFICATION.md` |
