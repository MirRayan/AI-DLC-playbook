# Guide 11.11 — Add a New Feature or Module (Mid-Project)

> **Time:** 2–4 hours | **Owner:** PM + dev lead + architect (as needed) | **Prerequisites:** Project at Phase 5 or 6

---

## Steps

| Step | Action | Owner | Verification |
|------|--------|-------|--------------|
| 1 | Add FR entries (new numbered requirements with stable [FR IDs](../glossary.md#fr-ids) — never renamed once published) to `docs/02-specification/SPECIFICATION.md`. Non-technical PMs: edit via the repo host's web editor or ask your AI assistant — see the [repo primer](../00-repo-basics.md) | PM | Acceptance criteria testable |
| 2 | Review for architectural impact — new ADR if needed | Architect | ADR Accepted or N/A confirmed |
| 3 | Create `docs/05-breakdown/modules/{NN}-{name}.md` from module template | Dev lead | Metadata + tickets assigned |
| 4 | If APIs involved: add `api:*` rows to `BACKEND-INDEX.md` (status `pending` until verified) | Dev / architect | IDs assigned |
| 5 | Add module row to `PROJECT-INDEX.md` and update `SPRINT-PLAN.md` | Dev lead + PM | Sprint assignment clear |
| 6 | First implementation session: [11.03 — Run an AI session](11.03-run-ai-session.md) with new module doc loaded | Developer | Context load succeeds |

---

## AI assist prompt — new module from spec

```markdown
Load the new FR entries for [feature name] from docs/02-specification/SPECIFICATION.md.
Load the module breakdown template.

Create docs/05-breakdown/modules/[NN]-[name].md with implementable tickets.
Assign next available [MODULE]-NNN IDs.
List api:* dependencies as pending if endpoints not yet verified.
```

---

## Checklist before first dev session

- [ ] FR IDs exist and are stable
- [ ] Module breakdown file exists
- [ ] Sprint plan updated
- [ ] PROJECT-INDEX module table updated
- [ ] API registry rows added (if applicable)
- [ ] Architect sign-off on ADR impact (if applicable)
