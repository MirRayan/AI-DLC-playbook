## Exit gate

The [exit gate](../glossary.md#exit-gate) for Phase 5: the first [sprint's](../glossary.md#sprint) [modules](../glossary.md#module) have implementable [tickets](../glossary.md#ticket) with IDs. A sprint plan exists. The [API registry](../glossary.md#backend-index) is started (if HTTP APIs). Module docs reference [`api:*` IDs](../glossary.md#api-registry-ids), not raw URLs.

---

## Steps

| Step | Action | Output | Verification |
|------|--------|--------|--------------|
| 1 | Create `docs/05-breakdown/SPRINT-PLAN.md` ‚Äî sprints, milestones, module‚Üísprint map | Plan exists | PM + dev lead agree |
| 2 | Copy module template ‚Üí `docs/05-breakdown/modules/{NN}-{name}.md` per module | One file per module | Numbering aligns with spec |
| 3 | Per module: metadata table, layered tickets (Data/Domain/UI or equivalent), QA tickets | Ticket IDs assigned | Each ticket traceable to FR |
| 4 | **If HTTP APIs:** verify endpoints; add rows to `docs/05-breakdown/backend/BACKEND-INDEX.md` | Registry populated | Stable `api:*` IDs assigned |
| 5 | For verified APIs: create `docs/04-reference/{module}-api.md` from template | Contracts exist | Paths, payloads documented |
| 6 | Optional: create `docs/05-breakdown/sprints/S1/` with sprint-scoped ticket lists | Sprint scope clear | Matches SPRINT-PLAN |
| 7 | Optional: create `docs/05-breakdown/STORY-POINTS.md` linking to external tracker | Estimates documented | Jira/Linear keys if used |
| 8 | Update `PROJECT-INDEX.md` ‚Äî module table, phase ‚Üí 5 complete | Index current | Module count matches breakdown |

## Module template source

Use `templates/module-breakdown-template.md` in your [bootstrapped](../glossary.md#bootstrap) context repo ‚Äî the platform-neutral module template, copied there from the bundled template when you ran [11.01 ‚Äî Create a context repo](11.01-create-context-repo.md). The playbook keeps a browsable copy at [templates/module-breakdown-template.md](../templates/module-breakdown-template.md).

Mobile teams can use the mobile-specific variant instead: `templates/mobile-module-breakdown.md` (adds iOS/Android platform rows and Figma design tracking).

## Three-layer API documentation

| Layer | Location | Purpose |
|-------|----------|---------|
| Registry | `BACKEND-INDEX.md` | Readiness + stable IDs |
| Contract | `docs/04-reference/*.md` | REST paths, payloads for implementers |
| Deep context | `docs/05-breakdown/backend/{module}/` | Admin paths, edge cases, backend internals |

Module breakdown docs reference **`api:*` IDs only**.
