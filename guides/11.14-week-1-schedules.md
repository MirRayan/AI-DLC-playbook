# Guide 11.14 — Role-Specific Week 1 Schedules

> **Time:** 1 week | **Owner:** All roles | **Prerequisites:** [11.01 — Create a context repo](11.01-create-context-repo.md) or [11.02 — Join an existing project](11.02-join-existing-project.md)

Role naming: **Business architect** below is the same role called "Architect" elsewhere in the playbook. **QA engineers** follow the Developer schedule plus [11.15 — Playwright testing](11.15-playwright-testing.md).

---

## Developer — Week 1

| Day | Focus | Guides / docs |
|-----|-------|---------------|
| **Day 1** | Access, orientation, context load | [11.02 — Join project](11.02-join-existing-project.md), [11.03 — Run AI session](11.03-run-ai-session.md), [04-role-developer.md](../04-role-developer.md) |
| **Day 2** | Read assigned module breakdown; understand sprint scope | Module doc in `docs/05-breakdown/modules/`, `SPRINT-PLAN.md` |
| **Day 3** | First ticket — AI session + implementation | [11.03 — Run AI session](11.03-run-ai-session.md), [11.10 — Development](11.10-phase-6-development.md) |
| **Day 4** | Continue tickets; check API registry before integrations | `BACKEND-INDEX.md`, `docs/04-reference/` |
| **Day 5** | First docs-touching PR (if applicable) | [11.13 — PR docs & APIs](11.13-pr-docs-and-apis.md) |

**Week 1 success:** Context load passes; one ticket merged or in review; knows where module docs and API registry live.

---

## Project manager — Week 1

| Day | Focus | Guides / docs |
|-----|-------|---------------|
| **Day 1** | Project orientation | [11.02 — Join project](11.02-join-existing-project.md), [04-role-pm.md](../04-role-pm.md) |
| **Day 2** | Review or draft concept note | [11.05 — Concept note](11.05-phase-1-concept-note.md), `CONCEPT-NOTE.md` |
| **Day 3** | Review or draft specification | [11.06 — Specification](11.06-phase-2-specification.md), `SPECIFICATION.md` |
| **Day 4** | Align with dev lead on breakdown readiness | [11.09 — Breakdown](11.09-phase-5-breakdown.md), `SPRINT-PLAN.md` |
| **Day 5** | Update index milestones; confirm out-of-scope list | `PROJECT-INDEX.md` |

**Week 1 success:** Spec has stable FR IDs; sprint plan agreed; out-of-scope explicit.
