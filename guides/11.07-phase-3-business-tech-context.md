## ADR-001 typical content

- Platform choice (web, mobile native, cross-platform, backend language)
- Key integrations (payment, auth, analytics, CRM)
- Deployment target (cloud, on-prem, hybrid)
- Version constraints

## ADR-002 typical content

- Architecture pattern (MVVM + Clean, hexagonal, microservices, etc.)
- Layer boundaries and dependency rules
- Data flow (API, local cache, events)
- Cross-cutting concerns (logging, error handling, i18n)
