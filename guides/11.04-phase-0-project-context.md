# Guide 11.04 — Phase 0: Project Context

> **Time:** 2–4 hours | **Owner:** Architect + tech lead | **Prerequisites:** [11.01 — Create context repo](11.01-create-context-repo.md)

---

## Exit gate

Team can answer without opening code: *What stack? What integrations? What constraints? What is out of scope?*

## Brownfield codebase scan prompt (Step 3)

```markdown
Analyze the codebase at [path or repo URL].

Output a bullet summary for docs/00-project-context/PROJECT-CONTEXT.md covering:
- Project structure and main modules
- Architecture patterns in use
- Key dependencies and integrations
- Technical debt and pain points
- Areas that need documentation before AI-assisted changes
```

---

## AI-ASSISTANT-RULES — what to customize

Add rules specific to your project. Examples:

```markdown
## ALWAYS
- Follow [architecture pattern] per ADR-002
- Write tests for all new UseCases / services
- Use [language version] and [framework version]

## NEVER
- Commit secrets or credentials
- Bypass the repository layer for API calls
- Add dependencies without architect approval
```

Keep the built-in pointers to `AGENTS.md` and `docs/process/DOCUMENTATION-SYNC.md`.
