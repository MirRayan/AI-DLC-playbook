# Guide 11.02 — Join an Existing AI-DLC Project

> **Time:** ~45 min | **Owner:** New team member | **Prerequisites:** Access to context repo + code repo(s)

---

## What you will have when done

- Understanding of current project phase and priorities
- Role-specific reading completed
- Successful AI context-load test
- First task document identified
- Clarity on who owns which artifacts

---

## Steps

| Step | Action | Verification |
|------|--------|--------------|
| 1 | Get access to **context repo** and relevant **code repo(s)**; clone all | `git clone` succeeds for each |
| 2 | Open `PROJECT-INDEX.md` — read **Quick Facts**, **Current Phase**, **Current Work Status** | You can state project phase and top 3 priorities aloud |
| 3 | Complete your **role reading list** (below) | Checklist done |
| 4 | Run context-load test per [11.03 — Run AI session](11.03-run-ai-session.md) | AI confirms correct project name, phase, constraints |
| 5 | Identify your **first task document** (module breakdown, spec section, or ADR) | File path written down |
| 6 | Ask doc DRI: *"What should I update when I change X?"* | You know index vs spec vs registry ownership |

---

## Step 3 — Role reading lists

### Developer

| # | File | Why |
|---|------|-----|
| 1 | `AI-SESSION-GUIDE.md` | How to work with AI on this project |
| 2 | `CONTEXT-RECOVERY.md` | What to load and when |
| 3 | `.ai/AI-ASSISTANT-RULES.md` | ALWAYS/NEVER rules for code |
| 4 | `docs/05-breakdown/modules/` — your assigned module | What to build |
| 5 | `docs/05-breakdown/backend/BACKEND-INDEX.md` (if APIs) | API readiness |

### Project manager

| # | File | Why |
|---|------|-----|
| 1 | `PROJECT-INDEX.md` | Current status and milestones |
| 2 | `docs/01-concept/CONCEPT-NOTE.md` | Vision and scope |
| 3 | `docs/02-specification/SPECIFICATION.md` | Canonical requirements (`FR-*`) |
| 4 | `docs/05-breakdown/SPRINT-PLAN.md` (if exists) | Release plan |
| 5 | `docs/05-breakdown/STORY-POINTS.md` (if exists) | Estimation context |

### Business architect

| # | File | Why |
|---|------|-----|
| 1 | `PROJECT-INDEX.md` | Decisions, ADR summary, status |
| 2 | `docs/03-context/BUSINESS-TECH-CONTEXT.md` | Business ↔ tech bridge |
| 3 | `docs/03-architecture/decisions/README.md` | ADR index |
| 4 | `docs/00-project-context/PROJECT-CONTEXT.md` | Integrations and constraints |
| 5 | `docs/05-breakdown/backend/BACKEND-INDEX.md` (if APIs) | API landscape |

---

## Step 6 — Ownership cheat sheet

Ask your DRI to confirm; typical defaults:

| When you change… | Update… | Owner |
|------------------|---------|-------|
| Project status / phase | `PROJECT-INDEX.md` | DRI + milestone owner |
| Requirements | `docs/02-specification/SPECIFICATION.md` | PM |
| Architecture decision | New ADR + ADR index | Architect |
| API contract | `BACKEND-INDEX.md` + `docs/04-reference/` | Dev / architect |
| Implementation tickets | `docs/05-breakdown/modules/` | Dev lead |
| Code | Code repo | Developer |
