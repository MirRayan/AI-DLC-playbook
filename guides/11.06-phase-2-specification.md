# Guide 11.06 — Phase 2: Specification

> **Time:** 1–3 days | **Owner:** PM + architect review | **Prerequisites:** [11.05 — Phase 1](11.05-phase-1-concept-note.md)

---

## Exit gate

The [exit gate](../glossary.md#exit-gate) for Phase 2: every in-scope feature has a **stable requirement ID** ([FR ID](../glossary.md#fr-ids)) and **testable [acceptance criteria](../glossary.md#acceptance-criteria)**. Out-of-scope items from the concept note remain marked OUT OF SCOPE.

---

## Steps

| Step | Action | Output | Verification |
|------|--------|--------|--------------|
| 1 | Copy [templates/SPECIFICATION-template.md](../templates/SPECIFICATION-template.md) → `docs/02-specification/SPECIFICATION.md` | File exists | ID scheme documented at top |
| 2 | Define ID scheme — e.g. `FR-{NNN}` or `FR-{MODULE}-{NNN}` — document at top of file | Scheme stated | Team agrees on format |
| 3 | For each in-scope feature from concept note: ID, description, acceptance criteria, priority | Requirements numbered | Every concept item has ≥1 FR |
| 4 | Mark deferred/out-of-scope items explicitly in spec | OUT OF SCOPE markers | Matches concept note |
| 5 | Optional: add traceability table (Concept item → FR IDs) | Table in spec or index | PM review |
| 6 | Architect reviews for technical feasibility | Comments resolved | No blocker requirements |
| 7 | Update `PROJECT-INDEX.md` — module list with FR counts, phase → 2 complete | Index current | Counts match spec |
| 8 | Save your work: commit and push (and PR if your team reviews doc changes) — see the [repo primer](../00-repo-basics.md) | Changes pushed | Remote updated |

## Requirement entry format

```markdown
### FR-HOME-001: Hero banner display

**Priority:** Must have | **Module:** Home

**Description:** The home screen displays a horizontally scrollable hero banner...

**Acceptance criteria:**
- [ ] Banner loads from configured API on app launch
- [ ] Tapping banner navigates to linked product/category
- [ ] Empty banner state shows fallback static image
- [ ] Banner respects user persona targeting rules

**Out of scope:** Banner admin CMS — managed via existing backend
```


---

## Common mistakes

| Mistake | Fix |
|---------|-----|
| Unstable IDs (renaming FR-001 later) | Assign final IDs now; never rename once referenced in breakdown |
| Vague acceptance criteria | Each criterion must be verifiable by QA |
| Skipping out-of-scope in spec | Copy from concept note; prevents accidental implementation |
