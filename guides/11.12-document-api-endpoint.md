# Guide 11.12 — Document a New API Endpoint

> **Time:** 30–60 min | **Owner:** Developer or architect | **Prerequisites:** Endpoint exists or is contractually agreed

---

## Rule

**Registry + reference + module refs must stay in sync.** Update all three in the same PR when an endpoint changes.

---

## Steps

| Step | Action | Verification |
|------|--------|--------------|
| 1 | Verify endpoint in target environment (curl, Postman, integration test, or backend repo) | Response matches expected shape |
| 2 | Assign stable `api:{domain}:{slug}` ID — **never rename once published** | ID follows existing naming pattern |
| 3 | Add row to `docs/05-breakdown/backend/BACKEND-INDEX.md` | Status set correctly |
| 4 | Create or update `docs/04-reference/{module}-api.md` — method, path, request, response, errors | Contract complete |
| 5 | In module breakdown, reference `api:*` ID only (not raw URL) | Module doc updated |
| 6 | Follow [11.13 — PR docs and APIs](11.13-pr-docs-and-apis.md) | Checklist complete |

---

## BACKEND-INDEX row format

| api:* ID | Method | Path | Status | Reference | Notes |
|----------|--------|------|--------|-----------|-------|
| `api:product:detail` | GET | `/V1/products/{sku}` | verified | [05-product-api.md](../04-reference/05-product-api.md) | Mobile PDP |

**Status values:**

| Status | Meaning |
|--------|---------|
| `verified` | Tested in target environment; safe to integrate |
| `pending` | Contract agreed; not yet verified in environment |
| `not_deployed` | Documented for future; do not implement yet |
| `deprecated` | Replaced; link to successor `api:*` ID |

---

## Reference doc section format

Use [api-reference-template.md](../AI-Driven-Project-Template/templates/api-reference-template.md) from the bundled template.

Minimum sections per endpoint:
- `api:*` ID
- Method and path
- Auth requirements
- Request parameters / body
- Response schema (with example)
- Error codes
- Mobile/client integration notes (if applicable)

---

## Verify with curl (example)

```bash
curl -s -X GET "https://api.example.com/V1/products/SKU123" \
  -H "Authorization: Bearer $TOKEN" | jq .
```

Document actual response shape in reference doc.
