# Guide 11.01 — Create a New Context Repo

> **Time:** ~30 min | **Owner:** Tech lead or architect | **Prerequisites:** [01-start-here.md](../01-start-here.md), the bundled [AI-Driven-Project-Template](https://github.com/MirRayan/AI-DLC-playbook/blob/main/AI-Driven-Project-Template/README.md) ("bundled" = it ships as a folder inside this playbook repo; clone or download the playbook to get it). This guide uses terminal commands and Git — if either is new to you, read [Repos, Git, and Terminals — a Primer for Everyone](../00-repo-basics.md) first.

---

## What you will have when done

- A new context repo bootstrapped from the template
- Git remote configured and initial commit pushed
- Code repo(s) checked out as siblings (not nested inside context repo)
- Team knows where to find this playbook

---

## Steps

| Step | Action | Verification |
|------|--------|--------------|
| 1 | Confirm **project name**, **code repo layout** (single vs multi-repo), and **doc DRI** (who owns `PROJECT-INDEX.md`) | Written in team channel or project charter |
| 2 | Clone or open `AI-Driven-Project-Template` locally | Template repo accessible |
| 3 | Run init script (see commands below) | New folder `../{kebab-project-name}/` created |
| 4 | `cd` into new project; open generated `QUICK-START.md` | File exists and lists next steps |
| 5 | Create the remote repo — the hosted copy on GitHub/GitLab/Azure DevOps. No terminal needed: use the host's website ("New repository" button) | Empty remote ready |
| 6 | Connect and upload: `git remote add origin <url>` then `git push -u origin main` (tells Git where the hosted copy lives, then uploads the initial commit) | `git remote -v` shows origin; remote has commit |
| 7 | Clone/checkout code repo(s) as **siblings** next to context repo (`git clone <url>` from the parent folder) | Folder layout matches diagram below |
| 8 | Share link to this playbook (`AI-DLC_Playbook/README.md`) with the team | Team can find onboarding docs |

### Step 3 — Commands

What each command does: `cd` moves into a folder; `chmod +x` makes the script runnable (Mac/Linux only, first time); `./init-project.sh` runs it.

**Mac/Linux:**

```bash
cd /path/to/AI-Driven-Project-Template
chmod +x init-project.sh
./init-project.sh "My Project"
mv ../my-project ../my-project-architecture   # add the -architecture suffix (naming convention)
cd ../my-project-architecture/
```

**Windows (PowerShell):**

```powershell
cd C:\path\to\AI-Driven-Project-Template
.\init-project.ps1 -ProjectName "My Project"
Rename-Item ..\my-project my-project-architecture   # add the -architecture suffix
cd ..\my-project-architecture\
```

The script names the new folder after your project (lowercased, hyphenated). Context repos are conventionally named `{project}-architecture`, so the rename step adds that suffix.

**Prefer not to touch a terminal?** Ask an [agentic AI tool](../glossary.md#agentic-tool) to run these steps for you, or follow the manual fallback in the [FAQ](../09-faq.md#init-projectsh-failed--what-now) — it works without the script.

The script:
- Copies template structure to `../{kebab-name}/`
- Replaces placeholders with your project name
- Creates empty phase directories
- Generates `QUICK-START.md`
- Runs `git init` + initial commit
- Removes init scripts from the new project

## Folder layout after Step 7

```
parent-folder/
├── my-project-architecture/    # context repo (docs + .ai) — THIS repo
├── my-project-api/             # optional code repo
├── my-project-web/             # optional code repo
└── my-project-mobile-ios/      # optional additional code repos
```

**Rule:** Application source code does **not** live inside the context repo. The context repo holds documentation, AI rules, and project metadata only.

---

## What init does NOT do

Init does **not** populate these mandatory files — you create them in the next guide:

- `PROJECT-INDEX.md`
- `.ai/context/project-overview.md`

Use the [playbook templates](../templates/README.md) when completing [11.04 — Phase 0: project context](11.04-phase-0-project-context.md).
