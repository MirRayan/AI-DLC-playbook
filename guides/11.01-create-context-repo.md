# Guide 11.01 — Create a New Context Repo

> **Time:** ~30 min | **Owner:** Tech lead or architect | **Prerequisites:** [01-start-here.md](../01-start-here.md), the bundled [AI-Driven-Project-Template](../AI-Driven-Project-Template)

---

## What you will have when done

- A new context repo bootstrapped from the template
- Git remote configured and initial commit pushed
- Code repo(s) checked out as siblings (not nested inside context repo)
- Team knows where to find this playbook

---

## Steps

| Step | Action | Verification |
|------|--------|--------------|
| 1 | Confirm **project name**, **code repo layout** (single vs multi-repo), and **doc DRI** (who owns `PROJECT-INDEX.md`) | Written in team channel or project charter |
| 2 | Clone or open `AI-Driven-Project-Template` locally | Template repo accessible |
| 3 | Run init script (see commands below) | New folder `../{kebab-project-name}/` created |
| 4 | `cd` into new project; open generated `QUICK-START.md` | File exists and lists next steps |
| 5 | Create remote repo (GitHub/GitLab/Azure DevOps) | Empty remote ready |
| 6 | `git remote add origin <url>` and push initial commit | `git remote -v` shows origin; remote has commit |
| 7 | Clone/checkout code repo(s) as **siblings** next to context repo | Folder layout matches diagram below |
| 8 | Share link to this playbook (`AI-DLC_Playbook/README.md`) with the team | Team can find onboarding docs |

### Step 3 — Commands

**Mac/Linux:**

```bash
cd /path/to/AI-Driven-Project-Template
chmod +x init-project.sh
./init-project.sh "My Project Name"
cd ../my-project-name/
```

**Windows (PowerShell):**

```powershell
cd C:\path\to\AI-Driven-Project-Template
.\init-project.ps1 -ProjectName "My Project Name"
cd ..\my-project-name\
```

The script:
- Copies template structure to `../{kebab-name}/`
- Replaces placeholders with your project name
- Creates empty phase directories
- Generates `QUICK-START.md`
- Runs `git init` + initial commit
- Removes init scripts from the new project

---

## Folder layout after Step 7

```
parent-folder/
├── my-project-architecture/    # context repo (docs + .ai) — THIS repo
├── my-project-api/             # optional code repo
├── my-project-web/             # optional code repo
└── my-project-mobile-ios/      # optional additional code repos
```

**Rule:** Application source code does **not** live inside the context repo. The context repo holds documentation, AI rules, and project metadata only.

---

## What init does NOT do

Init does **not** populate these mandatory files — you create them in the next guide:

- `PROJECT-INDEX.md`
- `.ai/context/project-overview.md`

Use templates from [templates/](../templates/) when completing [11.04 — Phase 0](11.04-phase-0-project-context.md).
