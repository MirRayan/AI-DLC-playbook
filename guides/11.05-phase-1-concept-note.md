# Guide 11.05 — Phase 1: Concept Note

> **Time:** 2–3 hours | **Owner:** PM (architect reviews) | **Prerequisites:** [11.04 — Phase 0](11.04-phase-0-project-context.md)

---

## Exit gate

Scope boundaries are explicit. An **out-of-scope** list exists. Stakeholders have reviewed vision and risks.

---

## Steps

| Step | Action | Output | Verification |
|------|--------|--------|--------------|
| 1 | Copy [templates/CONCEPT-NOTE-template.md](../templates/CONCEPT-NOTE-template.md) → `docs/01-concept/CONCEPT-NOTE.md` | File exists | Template placeholders replaced |
| 2 | Write: vision, target users, in-scope features, **out-of-scope**, success metrics, risks | All sections filled | PM self-review |
| 3 | Run AI review prompt (below) | Gaps addressed | No contradictions with Phase 0 |
| 4 | Architect reviews for technical feasibility | Comments resolved | Sign-off noted |
| 5 | Stakeholder review (meeting or async) | Approval recorded | Date + approver in doc or index |
| 6 | Update `PROJECT-INDEX.md` — vision summary, phase → 1 complete | Index current | Module/feature list started |

---

## AI review prompt (Step 3)

```markdown
Load docs/01-concept/CONCEPT-NOTE.md and docs/00-project-context/PROJECT-CONTEXT.md.

Review the concept note for:
- Gaps between stated vision and project context
- Contradictions with known constraints
- Missing out-of-scope items (what we are NOT building)
- Unrealistic success metrics
- Unaddressed risks

List findings as numbered items with suggested fixes.
```

---

## Out-of-scope examples

Be explicit. Good out-of-scope entries:

- "Admin panel — existing web console used; not in mobile scope"
- "Password-based login — OTP only per business rule"
- "Offline checkout — read-only catalog offline only"
