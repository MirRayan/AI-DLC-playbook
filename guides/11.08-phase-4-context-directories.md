# Guide 11.08 — Phase 4: Context Directories (Optional)

> **Time:** 2–4 hours | **Owner:** Architect + leads | **Prerequisites:** [11.07 — Phase 3](11.07-phase-3-business-tech-context.md)

---

## Skip this phase if

- Small project or single domain
- Tier-1 hot files (`PROJECT-INDEX.md`, `project-overview.md`, `AI-ASSISTANT-RULES.md`) are sufficient
- Team is fewer than ~5 people with one shared context

---

## Exit gate

Each major domain has an AI-loadable context slice. Domain files link to FR IDs and integration points.

---

## Steps

| Step | Action | Output | Verification |
|------|--------|--------|--------------|
| 1 | List domains from spec modules — e.g. auth, catalog, payments, notifications | Domain list | In PROJECT-INDEX or README |
| 2 | Choose location: `docs/04-context-directories/` and/or `.ai/context/features/` | Path decided | Team agrees |
| 3 | Create one file per domain — e.g. `auth-context.md`, `catalog-context.md` | Files exist | One per major domain |
| 4 | Per file: entities, business rules, integration points, links to FR IDs, key ADRs | Content complete | AI can load single domain |
| 5 | Update `PROJECT-INDEX.md` — context directory links | Index current | Links work |

---

## Domain context file template

```markdown
# Domain: Authentication

**FR refs:** FR-AUTH-001, FR-AUTH-002, ...
**ADRs:** ADR-002, ADR-004
**Integrations:** OTP provider, social login (Google, Facebook)

## Entities
- User, Session, OTPChallenge

## Business rules
- OTP expires after 5 minutes
- Guest browsing allowed without login
- JWT persisted in secure storage

## API dependencies
- api:auth:otp-send (verified)
- api:auth:otp-verify (pending)

## Out of scope
- Password reset flow — deferred to Phase 2 release
```

---

## When to load domain context

Add to AI session prompt when working on that domain:

```markdown
Also load docs/04-context-directories/auth-context.md
```

See [06-daily-workflows.md](../06-daily-workflows.md) — Task tier (5–10 min).
