## Steps (per ticket)

| Step | Action | Verification |
|------|--------|--------------|
| 1 | Start AI session — [11.03 — Run an AI session](11.03-run-ai-session.md) | Context loaded |
| 2 | Open module doc; pick ticket (e.g. `HOME-003`) | Ticket ID and acceptance criteria clear |
| 3 | Check `BACKEND-INDEX.md` for each `api:*` dependency | Status is `verified` or team accepts `pending` risk |
| 4 | Load API reference from `docs/04-reference/` if integrating | Contract understood |
| 5 | State intent; get plan approval for substantive work — [plan-before-act](../glossary.md#plan-before-act): the AI proposes its plan, **you** approve or correct it before it edits anything ([AI Assistant Basics](../00-ai-assistant-basics.md)) | Plan approved by you |
| 6 | Implement in **code repo** | Tests pass; lint clean |
| 7 | Mark ticket done in module doc (or external tracker) | Status updated |
| 8 | If API/ADR/spec changed: [11.13 — PR docs and APIs](11.13-pr-docs-and-apis.md) | Docs PR merged with code |

## Sprint boundary checklist

Update in `PROJECT-INDEX.md`:

- Current Work Status (what's done, in progress, blocked)
- Last Updated date
- API registry summary if readiness changed
- Any new pending decisions

---

## Mechanical verification

Before merging large documentation changes, run the scripts shipped in your context repo's `scripts/` folder:

```bash
./scripts/check-doc-links.sh          # internal links (requires lychee)
./scripts/verify-project-docs.sh      # full sweep — extend section 3 of the script (not lifecycle Phase 3) per project
```

`lychee` is a link-checking tool — install it with `brew install lychee` (Mac) or see its docs for other platforms. Not comfortable running scripts? Ask your [agentic AI tool](../glossary.md#agentic-tool) to run them and report the results.

Your project may rename the verify script (e.g. `verify-architecture-docs.sh`). See `scripts/README.md` in your context repo.

## When to escalate to architect

- New integration or third-party service
- ADR-002 pattern violation needed
- API contract change affecting multiple modules
- Cross-repo breaking change

Use `.ai/workflows/decision-making-protocol.md` in your context repo.
